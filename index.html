ColorfuLS

ColorfuLS は、色付きのファイル一覧をターミナルに出力する Rust 製 CLI ツールです。README.md の内容に加えて、現在の仕様、チュートリアル、実行例、内部仕様をまとめています。

現在の仕様

  • cols コマンドでカレントディレクトリの一覧を表示します。
  • color_rules プロファイルを読み込み、ファイル名やディレクトリ名に応じて色を決定します。
  • -a / --all で隠しファイルを含めて表示します。
  • -l / --long で詳細情報を表示します。
  • ディレクトリ、シンボリックリンク、実行可能ファイルを自動判定して色分けします。
  • Unix ではユーザー名とグループ名を表示し、Windows では数値 UID/GID にフォールバックします。
  • 色指定は 16 進、rgb、ansi、色名、既存の ANSI エスケープ列に対応します。

README.md の要点

README.md では次の点が案内されています。

  • ColorfuLS はディレクトリやファイル、拡張子ごとに色を設定してリスト表示できます。
  • プロファイル設定により、色付けルールを外部化できます。
  • コマンドは cols [OPTIONS] [FILE] の形式で実行します。

インストール

Rust と Cargo があれば、次のコマンドでビルドできます。

cargo build --release

実行ファイルは target/release/cols に生成されます。

使い方

cols [OPTIONS] [FILE]
オプション説明
-a / --all隠しファイルを含むすべてのエントリを表示します。
-l / --long詳細表示を行い、権限、リンク数、所有者、グループ、サイズ、更新時刻を表示します。
--profileプロファイルを開くために使用します。

チュートリアル

  1. リポジトリをクローンします。
  2. cargo build --release を実行してビルドします。
  3. ./target/release/cols を実行して一覧を表示します。
  4. ./target/release/cols -a で隠しファイルも表示します。
  5. ./target/release/cols -l で詳細表示を確認します。
  6. color_rules を編集して、独自の配色を試します。

COLOR_RULES 環境変数を設定すると、プロファイルの読み込み先を切り替えられます。

実行例

1. デフォルト表示

$ ./target/release/cols
README.md
Cargo.toml
src

2. 隠しファイルを含む表示

$ ./target/release/cols -a
.coverage
Cargo.toml
README.md
src

3. 詳細表示

$ ./target/release/cols -l
-rw-r--r--  1 user group  123 Apr 12 14:15 README.md
drwxr-xr-x  3 user group 4096 Apr 12 13:50 src

4. プロファイル例

dir:.*README.* => magenta
file:^.*\.rs$ => \x1b[32m
\.md$ => \x1b[33m
\.#.* => \x1b[90m

この設定では README をマゼンタ、Rust ファイルを緑、Markdown ファイルを黄色、隠しファイルを暗い灰色で表示します。

内部仕様

  • parse_rule_line がプロファイル行を解析します。
  • color_spec_to_escape が色指定を ANSI エスケープに変換します。
  • color_for_entry がルールとファイル種別から表示色を決定します。
  • format_long_entry_with_widths がロング表示の整形を担当します。
  • 関数単位のテストを追加して、返り値の妥当性を検証しています。
